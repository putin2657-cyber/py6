# PythonProject6 — OAIP (управление пользователями)

Проект реализует систему управления пользователями с двумя интерфейсами:

- Десктопное приложение на PyQt6 (папка `oaip/`)
- Веб-версия на Flask (папка `oaip_web/`)

Ниже — краткое руководство по установке, запуску и структуре проекта.

## Возможности

- Авторизация и регистрация пользователей
- Просмотр, редактирование и удаление пользователей
- Валидация логина и пароля (`oaip/validators.py`)
- Хранение данных в MySQL

## Требования

- Python 3.8+
- PyQt6 (для десктопной версии)
- Flask (для веб-версии)
- mysql-connector-python

В корне проекта есть `requirements.txt` для десктопной части. Для веб-части добавьте зависимости в `oaip_web/requirements.txt` (например: `Flask`, `mysql-connector-python`).

## Настройка базы данных

Проект использует MySQL. Для разработки можно создать базу и таблицу так:

```sql
CREATE DATABASE IF NOT EXISTS `uslugi` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE `uslugi`;
CREATE TABLE IF NOT EXISTS `user` (
  `id` INT AUTO_INCREMENT PRIMARY KEY,
  `login` VARCHAR(100) NOT NULL UNIQUE,
  `parol` VARCHAR(255) NOT NULL
);
```

Важно: не храните реальные пароли и секреты в репозитории. Используйте переменные окружения или .env для секретов.

## Установка и запуск

1) Создайте виртуальное окружение и установите зависимости (из корня проекта):

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2) Запуск десктопной версии:

```bash
python -m oaip.main
# или
python oaip/main.py
```

3) Запуск веб-версии (установите web-зависимости и запустите Flask):

```bash
# при активном виртуальном окружении
pip install -r oaip_web/requirements.txt
python3 -m flask --app oaip_web.app run
# или
python oaip_web/app.py
```

Сервер по умолчанию доступен по адресу: http://127.0.0.1:5000

## Валидация

Правила валидации (кратко):
- Логин: минимум 3 символа, обязательна строчная английская буква, без пробелов
- Пароль: минимум 6 символов, как минимум один спецсимвол

Подробности реализации — в `oaip/validators.py`.

## Безопасность и улучшения

- Пароли сейчас хранятся в открытом виде — для продакшена обязательно использовать хеширование (bcrypt или passlib).
- `app.secret_key` и параметры подключения к БД следует хранить в переменных окружения, не в коде.
- Рассмотреть добавление миграций (например, Alembic) и Docker-контейнера для БД при развёртывании.

## Структура проекта (кратко)

- `oaip/` — десктопная часть (PyQt6)
  - `main.py` — точка входа десктопа
  - `auth_form.py`, `reg_form.py` — формы авторизации/регистрации
  - `connect_db.py` — подключение к MySQL
  - `validators.py` — валидация

- `oaip_web/` — веб-приложение (Flask)
  - `app.py` — маршруты и логика
  - `templates/` — HTML-шаблоны
  - `static/` — CSS и статические файлы
